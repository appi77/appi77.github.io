OpenText Documentum xPlore CE 22.1 Installation Guide

✅ 설치 전 체크리스트

1. Linux 설정

  • 랜덤 생성기 활성화:
/sbin/rngd -b -r /dev/urandom -o /dev/random
  • GUI 설치 필요 (콘솔 모드 설치 불가)
  • 64비트 Linux에서는 다음 RPM 패키지 필요:
  • libXp
  • libXi
  • libXtst
  • libXt (버전, 릴리즈, 아키텍처는 OS에 맞게 선택)

2. Java 설치

  • Oracle JDK 11 또는 OpenJDK 11 설치 (공식 사이트에서 다운로드)

Installing the xPlore primary instance

1. 설치

  • Documentum_xPlore__windows_x64.zip
  • Documentum_xPlore__linux_x64.zip
  • Run setup.exe
  • ※ 반드시 primary xPlore instance와 Index Agent instance를 Create해야 "D:\xPlore\watchdog\config\dsearch-watchdog-config.xml" 에 반영이됨

2. Create the primary xPlore instance

  • Run xplore_home\setup\dsearch\configDsearch.bat (Windows) or xplore_home/setup/dsearch/configDsearch.sh (Linux)

3. Create the Index Agent instance

  • Run xplore_home\setup\indexagent\configIndexagent.bat (Windows)

  • 설치 시 오류 해결

    • 설치 시 global Registry password가 맞지 않다고 나오고 로그를 보면 [DFC_PREFERENCE_BAD_VALUE] Bad value for preference "dfc.date_format", value="yyyy. M. d. a h:mm:ss" 이 발생하면:
    • 방법 A: 시스템 로케일 변경 (가장 권장)
      1. 윈도우 제어판 > 시계 및 국가 > 국가 또는 지역으로 이동
      2. 형식(Format) 탭에서 '한국어'를 '영어(미국)'로 변경

✅ 설치 경로 규칙

1. 로컬 경로 사용

  • xPlore를 설치할 때는 Windows 로컬 드라이브 경로를 사용해야 합니다.
  • 예: F:\xPlore

2. 멀티 인스턴스 구성 시 UNC 경로 필요

  • 여러 xPlore 인스턴스가 동일한 데이터 및 설정 경로에 접근해야 하므로, UNC(Universal Naming Convention) 경로를 사용해야 합니다.
  • UNC 경로 형식:
\\<hostname>\F$\xPlore\data
\\<hostname>\F$\xPlore\config

3. 예시

  • 설치는 로컬 경로 F:\xPlore에 진행
  • Primary 인스턴스 설정 시, 데이터 및 설정 경로를 UNC로 지정:
데이터 경로: \\<hostname>\F$\xPlore\data
설정 경로: \\<hostname>\F$\xPlore\config

설치 후 확인

xPlore 접근 확인

예:

http://server.opentext.com:9300/dsearchadmin
http://server.opentext.com:9300/dsearch
http://server.opentext.com:9200/IndexAgent

Starting xPlore Administrator

http://<host:port>/dsearchadmin

Testing indexing

In xPlore Administrator, select Diagnostic and Utilities > Upload Testing Document.

Testing search (iAPI command)

?,c,SELECT r_object_id FROM dm_sysobject SEARCH DOCUMENT CONTAINS 'dmadin' ENABLE(FTDQL_DATA)

WatchDog 설정 및 문제 해결

WatchDog에서 DM_TICKET을 사용하기 위한 설정 방법

  1. $xplore_home/watchdog/config/dfc.properties 파일을 열거나, 없으면 새로 생성합니다.
  2. IndexAgent 설치 디렉터리의 dfc.properties 파일에서 다음 항목을 복사해 추가합니다.

패치

  1. mklink /d JAVA_LINK "D:\Java\jdk-17.0.17+10"
  2. update.exe LAX_VM ${xPlore_home}\java64\JAVA_LINK\bin\java.exe
