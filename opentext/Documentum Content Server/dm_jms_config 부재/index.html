Dm jms config 부재

dm_jms_config 객체 다시 생성
다른 모든 방법이 실패할 경우, dm_jms_config 객체를 다시 생성할 수 있습니다. 하지만 이러한 단계를 수행하기 전에 Documentum 기술 지원 엔지니어에게 문의하십시오.
최신 데이터베이스 백업 파일을 다운로드하세요.
기존 dm_jms_config 객체를 삭제합니다.

destroy,c,<dm_jms_config ID>
필요한 경우 dm_server_config에서 do_bpm 정보를 제거합니다.

이 경우 do_bpm은 배열 인덱스 2에서 발견되었습니다.  

remove,c,<dm_server_config ID>,app_server_name[2]
remove,c, <dm_server_config ID>,app_server_uri[2]
save,c, <dm_server_config ID>
reinit,c

dm_jms_config 객체를 다시 생성하려면
다음 dmbasic 프로그램을 사용할 수 있습니다.

$DM_HOME%\bin\dmbasic -f %DM_HOME%\install\admin\dm_jms_config_setup.ebs -e install -- <docbase> <install_owner acc> <install_owner passwd> <Content Server hostname> <port number>
bpm.dar 파일을 다시 설치하십시오.
BPM 기능을 실행 중이고 이전에 bpm.DAR 파일을 설치한 적이 있다면 bpm.dar 파일을 다시 설치하십시오. 이 DAR 파일 설치는 do_bpm 서블릿을 dm_jms_config 객체에 추가합니다.

실제 적용사례

%DM_HOME%\bin\dmbasic -f %DM_HOME%\install\admin\dm_jms_config_setup.ebs -e install -- EDMS ecmadmin Opentext1@ OTCM254 9080
